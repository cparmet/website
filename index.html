Pandas Checks: Open source package



I created an open source package, Pandas Checks, to support development and maintenance of data pipelines in Python. It's a little doctor's bag of tools to help data scientists, data engineers, and other data friends check the health of their data plumbing. These tools let you inspect and validate the results of individual steps of data processing on the fly, without needing to break apart and dirty up your functional code.

  It was my first open source package and I loved the new skills it taught me, including test automation, package documentation, release management, and GitHub automation.

  Pandas, method chains, functional programming, test automation (nox, pytest), GitHub Actions CI, package development (poetry, mkdocs, PyPi), precommit

Finite News: mindful, AI-assisted news



I'm happy to pay for subscriptions to quality news sources, but their websites are filled with so many distractions and attention vampires! So I created Finite News, an automated service that delivers the day's news without the noise. It uses a couple large language models (LLMs), real-time APIs, feeds, web scraping to gather and process content. A daily issue can report the latest happenings in your town, state, country, planet, and universe; forecast your local weather; give you a heads up on upcoming events in your area; track stocks of interest; give you last night's box scores (go Celtics üçÄ!); and get new content such as Webb telescope photos and XKCD comics. Rules and an LLM lasso the most informative content and filter out the noise.

  Finite News taught me the long-term benefits of functional programming. It was also my first time deploying to Google Cloud Platform.

  Python, Google Cloud Platform (Cloud Run Jobs, Cloud Build, Artifact Registry, Cloud Storage), AWS (Sagemaker, S3), HuggingFace sentence transformers, LLMs, OpenAI API, Selenium/Chromedriver, BeautifulSoup, Sendgrid, UV

Metadata classification



I created a hybrid system that assigns metadata to patient education documents. One piece of metadata is assigned by a machine learning model (text classifier) trained to identify 70 possible categories of documents. The other piece of metadata, with 6000 possible categories, is assigned through text processing and a live taxonomy.

  This project reminded me that we don't always need a machine learning model to solve our problem, and to always seek the simplest solution first. It's also when I began my love affair with MLFlow, used for tracking machine learning runs and experiments.

  Machine learning (Scikit-learn, MLFlow, random forests, logistic regression), NLP & text mining (spaCy, Regex), Python (Pandas, Numpy), SOLR API, Excel
